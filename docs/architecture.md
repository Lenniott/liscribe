## Component вҖ” inside the Panel Layer

Each panel is a self-contained component:

**ScribePanel**
- HTML/CSS view (pywebview)
- ScribeBridge вҖ” JSвҶ”Python calls (recording controls, waveform, notes)
- ScribeController вҖ” orchestrates AudioService + ModelService

**TranscribePanel**
- HTML/CSS view
- TranscribeBridge
- TranscribeController вҖ” file input вҶ’ ModelService вҶ’ output

**DictatePanel** (floating, near cursor)
- HTML/CSS view (minimal: waveform + timer only)
- DictateBridge
- DictateController вҖ” hotkey state machine + AudioService + paste

**SettingsPanel**
- HTML/CSS view (tabbed: General, Models, Hotkeys, Deps, Help)
- SettingsBridge
- reads/writes ConfigService directly

**OnboardingPanel**
- HTML/CSS view (stepped wizard)
- OnboardingBridge
- calls real workflows for practice steps

**Shared services (not panels):**
- **AudioService** вҖ” wraps recorder.py; one instance, shared across panels
- **ModelService** вҖ” wraps transcriber.py; download, load, run
- **ConfigService** вҖ” wraps config.py; single source of config truth
- **HotkeyService** вҖ” pynput listener; fires callbacks to DictateController and ScribeController
